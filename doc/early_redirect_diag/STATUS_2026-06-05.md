# BRU early-redirect — status & remaining recovery work (2026-06-05)

**The #1 boot-IPC lever** (commit-only-redirect penalty = 18–29% of boot; IPC ceiling 1.55→1.9–2.2).
Enabling it (EXEC_PARTIAL_BRANCH_RECOVERY + BRU0/1_EARLY_REDIRECT_ENABLE @core_top:13-18) deadlocks
because the partial-flush backend recovery is incomplete across **multiple in-flight completion paths**.
Unifying root: *a SURVIVING (older-than-flush) in-flight op must be preserved across a PARTIAL flush;
only WRONG-PATH (younger-than-flush) ops may be cancelled/suppressed.* The **multiplier already does this
correctly** and is the fix template for every other unit.

## Fixes — done, identified, and the multiplier template

1. **Loop-break (DONE, needed):** rename gate at core_top:533 uses the REGISTERED quarantine
   `bru_redirect_quarantine_r` (was combinational `bru_redirect_quarantine`) → breaks the 5 comb-loops
   (lint 20→15). Byte-identical when early-redirect off.

2. **p1-retry preservation (DONE, VALIDATED — fixes CoreMark):** lsu.sv p1_retry drain guarded with
   `!flush_in.valid` (~lsu.sv:1922). A surviving port-1 same-line retry coinciding with a partial flush
   was dropped (request flush-suppressed + retry drained + load_issue_valid_r not latched) → same-line
   dual-load partner stalled the ROB head → deadlock. Guard holds the retry through the flush cycle.
   Byte-identical when off (a full_flush is killed by the branch above; partial flushes only exist with
   early-redirect on). **This alone makes CoreMark PASS.**

3. **LMB orphan (NEEDED for boot — re-add):** a surviving port-0 load-miss whose one-shot D-cache
   `fill_snoop` is missed (alloc lands >FILL_BYPASS_DEPTH=4 cyc after the fill when the flush delays
   re-issue) is orphaned (valid, ready=0, no MSHR) → never completes. A timeout re-probe (re-issue the
   line via the load_req port; memory-safe because L1 is write-through + the load already cleared
   SQ-forward at issue) was built (~210 lines, gated, lint 15, validated to engage: fires=1 hits=1) then
   REMOVED as CoreMark-redundant — but the **boot DEADLOCKS with LMB orphans**, so it (or a cleaner
   p0-retry-preservation root fix analogous to #2) is REQUIRED. Spec: lmb_reprobe_fix_spec.md. Re-buildable
   from this session's transcript.

4. **Divider (IDENTIFIED — mirror the mul):** core_top:3685 `divider.flush(flush_out.valid)` cancels the
   in-progress divide on ANY flush, and core_top:3668 clears `div_rob_idx_r` on any flush → a surviving
   divide is killed by a partial flush → timeout (compliance div/divu/divw/rem/remw). FIX = mirror the mul
   (core_top:3644-3651): gate `.flush` to `flush_out.valid && flush_out.full_flush`, gate the
   div_rob_idx_r clear to full_flush, add `div_result_flushed = flush_out.valid && !full_flush &&
   rename_buf_partial_clear[div_rob_idx_r]` and `div_valid_live = div_valid_out && !div_result_flushed`,
   and consume div_valid_live at the CDB-arbitration sites (~core_top:2360, 3971, 3976).

5. **FPU (IDENTIFIED — same class):** fpu_top @core_top:2882 — compliance fadd/fcvt/fclass/fmove/fdiv/
   fmadd/recoding/structural TIMEOUT with early-redirect on. Inspect fpu_top's flush handling; apply the
   same survivor-aware pattern (full_flush-only cancel + younger-than-partial result suppress).

## Lesson
Validate recovery on the TARGET workload (boot), not a single benchmark — CoreMark masked the LMB-orphan,
DIV, and FPU holes that the boot exposes. Each "new bug" is the same root (partial flush vs surviving op);
the multiplier is the in-tree correct template for all of them.
