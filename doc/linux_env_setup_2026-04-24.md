### dsim build/run recipe (Linux)

```bash
cd /home/jeremycai/agent-workspace/rv64gc-v2

# One-time per change to RTL or testbench
./build_dsim.sh          # → dsim_work/tb_image.so + dsim_build.log

# Per benchmark / regression
./run_dsim.sh tests/hex/rv64ui_minimal.hex 5000
./run_dsim.sh tests/hex/dhrystone.hex      200000
./run_dsim.sh tests/hex/coremark.hex       500000  +PERF_PROFILE +STAT_DUMP
```

Both scripts source `shell_activate.bash` themselves; you don't need to
preload the DSim env in your shell. Override `DSIM_HOME` /
`DSIM_LICENSE` env vars before calling if your install moves.

   `.bat` runner shims. Needs a dsim backend if we want regression-driver

   `mingw32-make`, `wslpath`. The fallbacks resolve on Linux but the
   xsim section uses `.bat` callouts that won't run. Verilator targets
   should still build.
